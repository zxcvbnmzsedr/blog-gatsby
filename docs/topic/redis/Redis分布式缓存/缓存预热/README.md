缓存预热就是系统上线后，后者系统在重启的时候，将相关的缓存数据直接加载到Redis。
这样就可以避免在用户请求的时候，先查询数据库，然后再将数据缓存的问题，用户直接查询事先被预热的缓存数据。

　　解决：

1. 上线时加个接口，手动触发加载缓存，或者定时刷新缓存。
2. 数据量不大，可以在项目启动的时候自动进行加载。
